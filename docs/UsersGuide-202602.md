## 1. はじめに

### 1.1 RouteLoggerとは

RouteLoggerは、ウォーキングやハイキングの際にGPS位置と写真を記録するアプリです。国土地理院の地図上に現在地を表示し、移動経路を自動的に記録します。

### 1.2 主な機能

- **GPS追跡**: 移動経路を自動記録
- **写真撮影**: 現在地と撮影方向を紐付けて写真を保存
- **KMZエクスポート**: 記録データをKMZファイルとして保存（Google Earthで表示可能）
- **クラウド保存**: Firebaseに記録データをバックアップ（設定でON/OFF）
- **外部データ表示**: 他のアプリで作成したKMZ/GeoJSONを地図上に表示
- **オフライン対応**: 電波がなくても基本機能が使用可能

### 1.3 対応環境

- スマートフォン（iPhone、Android）のWebブラウザ
- Chrome、Safari推奨
- GPS機能が必要です

## 2. アプリの起動

### 2.1 初回アクセス

1. ブラウザでRouteLoggerのURLにアクセスします
2. 「位置情報の使用を許可しますか？」と表示されたら「許可」をタップします
3. 地図が表示され、現在地に緑の三角マーカーが表示されます

### 2.2 許可が必要な機能

| 機能 | 許可内容 |
|------|---------|
| GPS追跡 | 位置情報の使用 |
| 写真撮影 | カメラの使用 |
| 方向表示 | モーションセンサーの使用（iOSのみ） |

## 3. 画面の見方

### 3.1 メイン画面

```
┌─────────────────────────┐
│ [10:30]                 │ ← 現在時刻（設定でON/OFF）
│ ステータス表示エリア     │ ← 現在の状態・座標を表示
├─────────────────────────┤
│                         │
│      地図表示エリア      │ ← 国土地理院地図
│      △ ← 現在地マーカー  │
│      ● ← 写真マーカー   │
│                         │
├─────────────────────────┤
│ Start Stop Photo Data ⚙ │ ← 操作ボタン
└─────────────────────────┘
```

### 3.2 操作ボタン

| ボタン | 色 | 機能 |
|--------|-----|------|
| **Start** | 緑 | GPS記録を開始 |
| **Stop** | 赤 | GPS記録を停止 |
| **Photo** | 青 | 写真を撮影（GPS記録中のみ有効） |
| **Data** | 紫 | データ管理メニューを表示/非表示 |
| **⚙** | グレー | 設定ダイアログを表示 |

### 3.3 データ管理パネル

**Data**ボタンをタップすると表示されます。

| ボタン | 色 | 機能 |
|--------|-----|------|
| **List** | オレンジ | 撮影した写真の一覧を表示 |
| **Size** | シアン | 記録データのサイズを表示 |
| **Save** | 緑 | データを保存（Firebase ON: クラウド保存 / OFF: KMZファイル） |
| **Reload** | 青 | データを読み込み（Firebase ON: クラウドから / OFF: ファイルから） |
| **Clear** | グレー | 表示データとローカルデータを初期化 |

> **Save/Reloadボタンの動作は設定の「Use Firebase」によって切り替わります。**

## 4. GPS記録を開始する

### 4.1 記録の開始手順

1. **Start**ボタン（緑）をタップします
2. 確認ダイアログが表示されます
   - 初回の場合: 「新規記録を開始」をタップ
   - 前回のデータがある場合:
     - 「データ初期化」: 前回のデータを消して新規開始
     - 「データ追記」: 前回のデータに追記

### 4.2 記録中の表示

- ステータスに「追跡中」と表示されます
- 画面上部に現在の緯度・経度・精度が表示されます
- 移動すると地図上に緑色の軌跡が描かれます

### 4.3 記録条件

GPSポイントは以下の条件で自動記録されます：

- **60秒以上経過した場合**
- **20m以上移動した場合**（GPSの誤差を超えた実際の移動のみ）

これにより、バッテリー消費を抑えながら効率的に記録します。

### 4.4 注意事項

- 記録中は画面がスリープしません
- バッテリー消費に注意してください
- 建物内や谷間ではGPS精度が低下する場合があります

## 5. 写真を撮影する

### 5.1 撮影手順

1. GPS記録中に**Photo**ボタン（青）をタップします
2. カメラ画面が表示されます
3. 被写体を画面に合わせます
4. 白い丸の**シャッターボタン**をタップします
5. 撮影した写真と確認画面が表示されます

### 5.2 撮影後の操作

| 操作 | 説明 |
|------|------|
| **方向ダイアル**をドラッグ | 撮影角度を設定（-180°〜+180°、10°単位） |
| **◄ ►** ボタン | 角度を10°ずつ変更 |
| **Forward / Backward** ボタン | 進行方向の前方/後方を選択してタップすると保存 |
| **Text** | 写真にメモを残す |
| **Retake** | 撮り直す（現在の編集内容は破棄） |
| **✕** | カメラ画面を閉じる |

### 5.3 方向の設定と保存

撮影後に**方向ダイアル**で角度を調整し、**Forward**または**Backward**ボタンをタップすると写真が保存されます。

| 設定値 | 意味 |
|--------|------|
| 角度 0°・Forward | 進行方向の正面を撮影 |
| 角度 +90°・Forward | 進行方向の右側を撮影 |
| 角度 -90°・Forward | 進行方向の左側を撮影 |
| 角度 0°・Backward | 後方の正面を撮影 |

選択した角度に応じた矢印が写真の下部にスタンプされます。

**ポイント:** 間違った方向で保存しても、再度ダイアルを調整してボタンをタップすれば**同じ写真を上書き更新**できます（新しい写真は作成されません）。

### 5.4 保存後の状態

- 保存後もプレビュー画面は閉じないので、続けて方向を修正したりメモを追加したりできます
- 終了する場合は**✕**ボタンをタップしてください
- 地図上にオレンジ色の丸マーカーが追加（または更新）されます

### 5.5 写真の解像度

- 保存される写真のサイズ: 720×1280ピクセル（縦向き）
- ファイル形式: JPEG

## 6. GPS記録を停止する

### 6.1 停止手順

1. **Stop**ボタン（赤）をタップします
2. 「追跡を停止しました」と表示されます
3. 記録データが自動的にローカル保存されます

### 6.2 停止後の状態

- 画面のスリープ防止が解除されます
- 記録したデータはアプリ内に保持されます
- 次回Startで「データ追記」を選ぶと続きから記録できます

## 7. データを確認する

### 7.1 写真一覧を見る

1. **Data**ボタン → **List**ボタンをタップ
2. 撮影した写真がグリッド表示されます
3. 写真をタップすると拡大表示されます
4. 拡大画面では撮影日時・位置・方向角度・テキストメモが確認できます
5. 左右の矢印ボタンで前後の写真に切り替えられます

### 7.2 データサイズを確認する

1. **Data**ボタン → **Size**ボタンをタップ
2. 以下の情報が表示されます：
   - トラック件数と記録ポイント数
   - GPSデータのサイズ
   - 写真の枚数とサイズ

## 8. データを保存する

SaveボタンはSettings（⚙）の「**Use Firebase**」設定によって動作が変わります。

### 8.1 KMZファイルとして保存（Use Firebase: OFF）

Firebase設定がOFFの場合、KMZファイルとしてダウンロードできます。

1. **Data**ボタン → **Save**ボタンをタップ
2. ファイル名の入力画面が表示されます
   - デフォルト: `RLog-YYYYMMDD`（当日の日付、JST）
   - 任意の名前に変更可能
3. 「保存」をタップ
4. KMZファイルがダウンロードされます

**KMZファイルについて:**
- Google Earth、Google Mapsなどで開けます
- GPS軌跡と写真位置・画像が含まれます
- 保存中は画面がスリープしません

### 8.2 クラウドに保存（Use Firebase: ON）

Firebase設定がONの場合、クラウド（Firebase）にデータを保存します。

1. GPS記録を開始した後（Start後）
2. **Data**ボタン → **Save**ボタンをタップ
3. ドキュメント名の入力画面が表示されます
   - デフォルト: `RLog-YYYYMMDD`
   - 任意の名前に変更可能
4. 「保存」をタップ

**保存されるデータ:**
- GPS軌跡データ
- 撮影した写真（テキストメモ含む）

**注意事項:**
- インターネット接続が必要です
- 写真が多い場合は保存に時間がかかることがあります
- 保存中は画面がスリープしません
- 同じ名前で保存すると自動的に連番が付きます（例: RLog-20260228_2）

## 9. データを読み込む

ReloadボタンはSettings（⚙）の「**Use Firebase**」設定によって動作が変わります。

### 9.1 KMZ/GeoJSONファイルを読み込む（Use Firebase: OFF）

1. **Data**ボタン → **Reload**ボタンをタップ
2. ファイル選択ダイアログが表示されます
3. 読み込むファイルを選択します（対応形式: .kmz, .kml, .geojson, .json）

**RouteLoggerで作成したKMZの場合:**
- 確認ダイアログが表示されます
- 「OK」をタップすると現在のデータをクリアしてKMZのデータを復元します
- 復元後、ページが自動的に再読み込みされます

**他のアプリで作成したKMZ/GeoJSONの場合:**
- 外部レイヤーとして地図上に表示されます
- 現在の記録データには影響しません
- アプリを再起動しても表示が維持されます

### 9.2 クラウドから読み込む（Use Firebase: ON）

1. **Data**ボタン → **Reload**ボタンをタップ
2. 保存済みドキュメントの一覧が表示されます
3. 読み込みたいドキュメントをタップ
4. データが読み込まれ、地図上に軌跡と写真マーカーが表示されます

## 10. データを初期化する

### 10.1 初期化手順

1. **Data**ボタン → **Clear**ボタンをタップ
2. 確認ダイアログが表示されます
3. 「OK」をタップ

### 10.2 初期化される内容

- 地図上のルートとマーカーが消去されます
- ローカルデータ（IndexedDB）が初期化されます

> **注意:** クラウド（Firebase）に保存済みのデータは削除されません。ローカルのデータのみ削除されます。

## 11. 設定を変更する

### 11.1 設定の開き方

**⚙**（設定）ボタンをタップします。

### 11.2 設定項目

| 設定名 | 説明 | デフォルト |
|--------|------|---------|
| **Show Clock** | 地図上に現在時刻を表示する | ON |
| **Use Firebase** | Save/LoadをFirebase（クラウド）経由にする | OFF |
| **Show Facing Buttons** | 写真撮影時のForward/Backwardボタンを表示する | ON |

### 11.3 Use Firebaseについて

| 設定 | Saveボタン | Reloadボタン |
|------|-----------|------------|
| OFF | KMZファイルをダウンロード | ファイルを選択して読み込み |
| ON | Firebaseクラウドに保存 | Firebaseから読み込み |

## 12. ホーム画面に追加する

RouteLoggerはPWA（Progressive Web App）対応のため、ホーム画面にアプリとして追加できます。

### 12.1 iPhoneの場合

1. Safariでアプリを開きます
2. 画面下部の共有ボタン（□↑）をタップ
3. 「ホーム画面に追加」をタップ
4. 名前を確認して「追加」をタップ

### 12.2 Androidの場合

1. Chromeでアプリを開きます
2. 「ホーム画面に追加」のバナーが表示されたらタップ
3. または、メニュー（⋮）から「ホーム画面に追加」を選択

### 12.3 メリット

- ブラウザのアドレスバーなしで全画面表示
- ホーム画面からワンタップで起動
- 一部機能がオフラインでも使用可能

## 13. よくある質問

### Q1. GPS精度が悪いのですが？

**A:** 以下をお試しください：
- 建物の外に出る
- 空が開けた場所に移動する
- 端末の位置情報設定を「高精度」にする
- しばらく待つ（GPS補足に時間がかかる場合があります）

### Q2. 写真が保存されません

**A:** GPS記録中（Start後）でないと写真は撮影できません。Startボタンをタップしてから撮影してください。

### Q3. SaveボタンでKMZが保存されず、Firebaseに繋ごうとします

**A:** 設定（⚙）の「Use Firebase」がONになっています。OFFにすると、KMZファイルとして保存されます。

### Q4. KMZをGoogle Earthで開いたら写真が表示されない

**A:** KMZファイルには写真データが含まれています。Google Earthを最新版に更新してお試しください。

### Q5. クラウド保存ができません

**A:**
- インターネット接続を確認してください
- 設定の「Use Firebase」がONになっているか確認してください
- 写真のサイズが大きい場合、アップロードに時間がかかります

### Q6. 過去のデータが消えました

**A:**
- ブラウザのデータを削除すると、ローカルデータは消去されます
- 重要なデータは必ずSave（KMZまたはクラウド）してください

### Q7. バッテリーの消費が激しいです

**A:** GPS追跡中は位置情報を常に取得するため、バッテリー消費が増加します。長時間の記録時はモバイルバッテリーの携帯をおすすめします。

### Q8. 方向ボタン（Forward/Backward）が表示されません

**A:** 設定（⚙）の「Show Facing Buttons」がOFFになっています。ONにすると表示されます。

## 14. トラブルシューティング

### 14.1 位置情報が取得できない

1. ブラウザの位置情報許可を確認
2. 端末の位置情報サービスがONになっているか確認
3. 屋外の開けた場所で再試行

### 14.2 カメラが起動しない

1. ブラウザのカメラ許可を確認
2. 他のアプリがカメラを使用していないか確認
3. ページを再読み込み

### 14.3 画面が固まった

1. ページを再読み込み（更新）
2. ブラウザを一度閉じて再起動
3. 端末を再起動

### 14.4 データが読み込めない（Firebase）

1. インターネット接続を確認
2. 設定の「Use Firebase」がONか確認
3. しばらく待ってから再試行
4. 別のネットワーク（Wi-Fi/モバイル）で試す

### 14.5 KMZファイルが読み込めない

1. ファイルの拡張子が `.kmz`, `.kml`, `.geojson`, `.json` のいずれかか確認
2. ファイルが破損していないか確認
3. ブラウザを再読み込みして再試行
