## 1. はじめに

### 1.1 RouteLoggerとは

RouteLogger は、ハイキング・散歩・サイクリングなどで GPS 位置と写真を記録するためのアプリです。国土地理院の地図上に現在地を表示し、移動経路を自動的に記録します。

### 1.2 主な機能

- **GPS追跡**: 移動経路を自動記録（5秒以上の間隔・60秒経過 または 20m以上の有意な移動で記録）
- **写真撮影**: 現在地と撮影方向（角度＋Forward/Backward）を紐付けて保存
- **写真の編集**: 写真ビューアでメモの編集・方向の変更・削除が可能
- **KMZエクスポート**: 記録データを KMZ ファイルとして保存（Google Earth で表示可能）
- **クラウド保存**: Firebase に記録をバックアップし、登録メールアドレスへ KMZ を自動送信
- **外部データ表示**: 他アプリで作成した KMZ/KML/GeoJSON を地図上に重ね表示
- **箕面オーバーレイ**: 緊急ポイントや公式ハイキングルートを地図に重ね表示
- **メッセージ履歴**: ステータスバーに表示されたメッセージを後から確認可能
- **オフライン対応**: 地図タイルとデータのCloud保存以外はオフラインでも動作

### 1.3 対応環境

- スマートフォン（iPhone/Android）の Web ブラウザ
- Chrome / Safari 推奨
- 位置情報・カメラ・モーションセンサー（iOSのみ）の使用許可が必要

## 2. アプリの起動

### 2.1 初回アクセス

1. ブラウザで RouteLogger の URL にアクセス
2. 「位置情報の使用を許可しますか？」と表示されたら **許可**
3. 地図が箕面大滝（既定の中心地）に表示されます

### 2.2 許可が必要な機能

| 機能 | 許可内容 |
|------|---------|
| GPS追跡 | 位置情報の使用 |
| 写真撮影 | カメラの使用 |
| 方向表示 | モーションセンサーの使用（iOS 13以降は記録開始時に確認） |

## 3. 画面の見方

### 3.1 メイン画面

```
┌─────────────────────────┐
│        [10:30]          │ ← 現在時刻（Startボタン真上、設定でON/OFF）
│ ステータス・座標表示     │ ← 一定時間で自動非表示
├─────────────────────────┤
│                         │
│      地図表示エリア      │ ← 国土地理院標準地図
│      △ ← 現在地マーカー  │
│      ● ← 写真マーカー   │
│                         │
├─────────────────────────┤
│ Start Stop Photo Data ⚙ │ ← メイン操作ボタン
└─────────────────────────┘
```

### 3.2 メイン操作ボタン

| ボタン | 機能 | 主な無効条件 |
|--------|------|-----------|
| **Start** | GPS記録を開始 | 追跡中／保存・読込中 |
| **Stop** | GPS記録を停止 | 待機中／保存・読込中 |
| **Photo** | 写真を撮影 | 待機中／保存・読込中 |
| **Data** | データ管理パネルを表示／非表示 | 保存・読込中 |
| **⚙ Settings** | 設定ダイアログを開く | 保存・読込中 |

### 3.3 データ管理パネル

**Data** ボタンをタップすると下部に表示されます。

| ボタン | 機能 |
|--------|------|
| **Clear** | 表示データとローカルデータを初期化（外部レイヤーも対象） |
| **Photos** | 撮影写真／外部写真の一覧を表示（タブ切替） |
| **Size** | 記録件数・データサイズを表示 |
| **Save** | データを保存（Firebase ON: クラウド保存 / OFF: KMZファイル） |
| **Load** | KMZ/KML/GeoJSON ファイルを読み込む（クラウドからの読み込みは廃止） |

> **Save** ボタンは Settings の「Use Firebase」によって保存先が切り替わります。Load は常にファイルからの読み込みです。

## 4. GPS記録を開始する

### 4.1 記録の開始手順

1. **Start**（緑）をタップ
2. 既存データがある場合は確認ダイアログが表示されます
   - **Start New (Clear Data)**: 既存データを消して新規記録を開始
   - **Continue (Append)**: 既存トラックに追記する形で開始（合計点数が継続表示されます）
   - **Cancel**: 開始をやめる
3. 既存データがない場合は **Start** または **Cancel** のみ表示されます

### 4.2 記録中の表示

- ステータスに **「GPS記録中 (n点記録)」** と表示されます（n は合計点数）
- 画面上部に現在の緯度・経度・精度・直前記録からの移動距離・経過秒が表示されます
- 移動するとネイビー色の軌跡が描かれます（色・太さは詳細設定で変更可能）
- 地図は現在地を中心に自動追従します

### 4.3 記録条件

- 直前記録から **5秒以上** 経過していること（高頻度記録防止）
- かつ、以下のいずれかを満たすこと
  - 60秒以上経過した
  - 20m 以上 かつ 現在の精度を超える距離を移動した（GPSのブレを除外）

これによりバッテリー消費を抑えながら効率的に記録します。

### 4.4 注意事項

- 記録中は画面が自動でスリープしません（Wake Lock）
- バッテリー消費が増加します。長時間の記録時はモバイルバッテリーの携帯を推奨します
- 建物内や谷間ではGPS精度が低下します

## 5. 写真を撮影する

### 5.1 撮影手順

1. GPS記録中に **Photo**（青）をタップ
2. カメラ画面が開きます（背面カメラ、プレビュー）
3. 白い丸の **シャッターボタン** をタップ
4. 撮影直後にプレビュー＋方向設定画面が表示されます

### 5.2 撮影直後の自動保存

撮影直後は **角度 0°・Forward** がデフォルトとして自動保存されます。方向ボタンを操作しなくても、写真は失われません。

### 5.3 撮影後の操作

| 操作 | 説明 |
|------|------|
| **方向ダイアル**をドラッグ | 撮影角度を設定（-180°〜+180°、10°単位） |
| **◀ ▶** ボタン | 角度を10°ずつ変更 |
| **Forward** | 進行方向の前方を表すフラグをON/OFF |
| **Backward** | 進行方向の後方を表すフラグをON/OFF（Forwardと両方ONも可） |
| **Memo** | 写真にテキストメモを残す（メモ入力済みのときはボタンが強調表示） |
| **Retake** | 現在の写真を破棄してプレビュー画面に戻る |
| **✕** | カメラ画面を閉じる |

> 角度の変更や Forward/Backward のタップ、メモ更新は、その都度 IndexedDB に上書き保存され、地図上のマーカーも更新されます。プレビュー画面は連続編集できるよう自動で閉じません。

### 5.4 方向の例

| 設定値 | 意味 |
|--------|------|
| 0°・Forward | 進行方向の正面を撮影 |
| +90°・Forward | 進行方向の右側を撮影 |
| -90°・Forward | 進行方向の左側を撮影 |
| 0°・Backward | 後方の正面を撮影 |

選択した角度に応じた矢印が写真の下部にスタンプされます。

### 5.5 写真の解像度

詳細設定（[15. 画面設定等](#15-画面設定等詳細設定)）で以下の3段階から選択できます。
- 高: 720×1280 px
- 中: 360×640 px（既定）
- 低: 180×320 px

JPEG 品質は 60〜80%（既定 70%）で調整できます。

## 6. GPS記録を停止する

### 6.1 停止手順

1. **Stop**（赤）をタップ
2. 「GPS記録を停止しました (n点記録)」と表示されます

### 6.2 停止後の状態

- 画面のスリープ防止が解除されます
- 記録データは IndexedDB に保持されます
- 次回 Start で **Continue (Append)** を選ぶと続きから記録できます

## 7. 写真を見る・編集する

### 7.1 写真一覧（Photosボタン）

1. **Data** → **Photos** をタップ
2. 上部のタブで切り替えます
   - **撮影写真**: 自分で撮影した写真
   - **外部写真**: インポートした KMZ/GeoJSON に含まれる写真
3. サムネールをタップすると拡大表示されます

各サムネールには方向バッジ（角度＋矢印アイコン）が、メモがある場合はその下にメモ本文が表示されます。

### 7.2 撮影写真ビューア

ビューアでは次の操作ができます。

| ボタン／操作 | 説明 |
|------------|------|
| ◀ ▶（左右） | 前後の写真へ切り替え |
| ピンチイン／アウト・ホイール | 1〜10倍までズーム |
| ドラッグ | ズーム中の画像を移動 |
| **Forward / Backward** | 個別にON/OFF切り替え（両方ONも可）。タップすると即座に保存されます |
| **Edit Memo** | メモを編集（保存／キャンセル可、Ctrl+Enterで保存・Escでキャンセル） |
| **Delete** | 確認後に写真とマーカーを削除 |
| ✕ | ビューアを閉じる |

メモ編集中に他の写真へ切り替えたり閉じたりすると、保存／破棄を確認するダイアログが表示されます。

### 7.3 外部写真ビューア

外部写真は読み取り専用です。Forward/Backward・Edit Memo・Delete ボタンは表示されません。Google Drive のリンク付き写真はビューア内で元画像を表示できます。

### 7.4 写真ライトボックス（外部リンク表示）

外部写真の説明欄に含まれる Google Drive 等のリンク、または「元の写真を表示」リンクをタップすると、画面全体に元画像が表示されます。読み込みに失敗した場合は埋め込みプレビューに自動で切り替わります。一度表示した画像は内部にキャッシュされ、次回以降の表示が速くなります。

### 7.5 地図上の写真マーカー

オレンジ色の丸（既定はネイビー、色は詳細設定で変更可能）の写真マーカーをタップすると、その写真の拡大ビューアが開きます。マーカー作成後に方向やメモを変更しても、IndexedDB から最新データを取得して表示します。

## 9. データを保存する

Save の動作は Settings の「**Use Firebase（保存時にクラウドのデータベースを使用）**」によって変わります。保存先はドキュメント名入力画面のタイトルで確認できます。

### 9.1 KMZファイルとして保存（Use Firebase: OFF）

Firebase 設定が OFF のとき、KMZ ファイルとしてダウンロードできます。

1. **Data** → **Save** をタップ
2. **「Save to file as...」** というタイトルのダイアログが表示されます
   - デフォルト: `RLog-YYYYMMDD`（当日の日付、JST）
   - 名前は自由に変更可能
3. **Save** をタップ
4. ステータスに `ファイルに保存中: "ファイル名.kmz"...` と表示され、KMZ ファイルがダウンロードされます

**KMZファイルについて**
- Google Earth、Google Maps などで開けます
- GPS 軌跡と写真位置・画像が含まれます
- 保存中は画面がスリープしません

> 注意: トラックが1件もない場合は「保存するルートログがありません」と表示されます。

### 9.2 クラウドに保存（Use Firebase: ON）

Firebase 設定が ON のとき、クラウド（Firebase）にデータを保存します。**事前にユーザー名の登録が必要です**（[12.4 ユーザー接続](#124-ユーザー接続firebase-on時のみ表示)）。

1. **Data** → **Save** をタップ
2. ユーザー名が未登録の場合はアラートが出て、Settings ダイアログが自動で開きます
3. ユーザー名が登録済みなら **「Save to cloud as...」** のダイアログが表示されます
   - デフォルト: `RLog-YYYYMMDD`
   - 自由に変更可能
4. **Save** をタップ
5. ステータスに `クラウドに保存中: "ドキュメント名"...` → 写真アップロード進捗 → `Firebase保存完了` の順で表示されます
6. 同じ名前が既に存在する場合は自動的に連番（`name_2` など、最大100まで）が付与されます

**保存後の自動処理**
- サーバー側で KMZ ファイルが生成され、サムネール（既定 160px）が作成されます
- 写真とサムネール・KMZ は Google Drive にアップロードされます
- 登録メールアドレス宛に KMZ ファイルが添付されたメールが届きます
- 完了時はステータスに `KMZ送信完了 → メールアドレス` と表示されます

**注意事項**
- インターネット接続が必要です
- 写真が多い場合は時間がかかります
- 保存中は画面がスリープせず、メイン操作ボタンが一時的に無効になります

## 10. データを読み込む

**Load** ボタンは常にファイルから読み込みます。

### 10.1 KMZ/KML/GeoJSONファイルを読み込む

1. **Data** → **Load** をタップ
2. ファイル選択ダイアログが表示されます
3. 対応形式: `.kmz`, `.kml`, `.geojson`, `.json`, `.zip`

**RouteLogger で作成したデータの場合**
- 確認ダイアログが表示されます
- **OK** で現在のデータをクリアし、トラックと写真を復元
- 復元後、ページが自動的にリロードされます
- 復元データは地図上にネイビー色で表示されます

**他のアプリで作成したデータの場合**
- 外部レイヤーとして地図上に青色（`#0055ff`）で表示されます
- 現在の記録データには影響しません
- アプリを再起動しても表示が維持されます（外部データとして保存）
- KMZ に画像が含まれる場合は写真一覧の **外部写真** タブから閲覧できます

## 11. データを初期化する

### 11.1 初期化手順

1. **Data** → **Clear** をタップ
2. 確認ダイアログで **OK**

### 11.2 初期化される内容

- 地図上のルート・マーカー・外部レイヤーがすべて消去されます
- ローカルデータ（IndexedDB）が完全に初期化されます（撮影写真／外部写真／外部GeoJSONも対象）

> **注意:** クラウド（Firebase）に保存済みのデータは削除されません。ローカルのデータのみ削除されます。

## 12. 設定を変更する

### 12.1 設定の開き方

**⚙ Settings** をタップします。

### 12.2 通常設定項目

| 設定名 | 説明 | 既定 |
|--------|------|------|
| **時刻を表示** | 地図上に現在時刻を表示する | ON |
| **保存時にクラウドのデータベースを使用** | Save を Firebase（クラウド）経由にする | OFF |
| **進行方向ボタンを表示** | 写真撮影後に Forward/Backward ボタンを表示 | ON |
| **箕面緊急ポイントを表示** | 箕面の緊急ポイント（約170点）をマーカー表示 | ON |
| **ハイキングルート(公式)を表示** | 箕面のハイキングルートとスポットを表示 | OFF |

### 12.3 Use Firebase について

| 設定 | Save ボタン（ダイアログタイトル） | Load ボタン |
|------|--------------------------------|------------|
| OFF | KMZ ファイルをダウンロード（Save to file as...） | ファイルを選択して読み込み |
| ON | Firebase クラウドに保存（Save to cloud as...） + KMZ をメール送信 | ファイルを選択して読み込み（クラウド読み込みは廃止） |

### 12.4 ユーザー接続（Use Firebase ON時のみ表示）

クラウド保存にはユーザー登録が必要です。設定ダイアログ内の「ユーザー接続」エリアをタップすると編集フォームが開きます。

| 項目 | 入力ルール |
|------|----------|
| ユーザー名 | 半角英数字のみ（全角や記号は自動で除去・小文字化） |
| メールアドレス | 全角は自動で半角に変換 |
| 氏名 | 姓のみでも可 |
| 同意チェック | 「私は箕面ハイキングアプリ開発の協力者です。」へのチェックで [登録] が有効 |

**[登録]**: 既存ユーザーであればメールアドレスの一致確認後に氏名を更新、新規ユーザーであればユーザー登録を行います。
**[クリア]**: フォームと保存済み情報をすべて消します。
**[キャンセル]**: 編集をやめて元の表示に戻ります。

通常表示では `@<ユーザー名>` と「（ユーザー登録確認済み）」「（ユーザー登録なし）」のステータスが表示されます。

### 12.5 アプリバージョン

設定ダイアログ右下の「アプリバージョン」エリアには、現在動いている Service Worker のキャッシュ名（例: `RLog-v10.7`）が表示されます。

## 13. 箕面オーバーレイを表示する

### 13.1 箕面緊急ポイント

設定で **箕面緊急ポイントを表示** を ON にすると、約170点の緊急ポイントが地図にマーカー表示されます。マーカーをタップすると ID と名前がポップアップで表示されます。色や半径は[詳細設定](#15-画面設定等詳細設定)で変更できます。

### 13.2 ハイキングルート（公式）

設定で **ハイキングルート(公式)を表示** を ON にすると、箕面の公式ハイキングルートとスポットが表示されます。ルートはオレンジ色のラインで、スポットは青色の四角アイコンで描画されます。色・サイズは詳細設定で変更可能です。

---

## 14. メッセージ履歴を見る

ステータスバーに表示されたメッセージは、最大200件まで端末内に保存されます（GPS記録中の表示は履歴に残りません）。

1. **⚙ Settings** をタップ
2. 「メッセージ履歴」ボタンをタップ
3. 日時・アプリバージョン・本文が新しい順に表示されます
4. 「クリア」で全削除（確認あり）、「閉じる」で履歴ダイアログを閉じます

ステータスメッセージはメイン画面上部に約10秒間表示されたあと自動で消えます。後から振り返りたいときに便利です。

## 15. 画面設定等（詳細設定）

撮影写真の解像度や地図上のマーカー色などをカスタマイズしたい場合は、隠し詳細設定を使います。

### 15.1 詳細設定の開き方

設定ダイアログ右下の **アプリバージョン** ラベルを **3秒以内に5回連続タップ** すると、「画面設定等」セクションが表示されます。

### 15.2 画像解像度の設定

| 項目 | 範囲 | 既定 |
|------|------|------|
| 写真解像度 | 高（720×1280）／中（360×640）／低（180×320） | 中 |
| JPEG画像品質 | 60〜80%（10刻み） | 70% |
| サムネール | 80〜320 px（40刻み） | 160 px |

スライダーまたは数値入力で調整し、 **[設定]** で保存、**[規定値]** で既定値に戻し、**[キャンセル]** で破棄します。サムネールサイズはクラウド保存後にメール送信される KMZ にも反映されます。

### 15.3 マーカーの設定

| 対象 | 既定の色 | 既定のサイズ |
|------|---------|-------------|
| 緊急ポイント | `#00AA00`（緑） | 半径 7 px |
| ハイキングルート | `#FF8C00`（オレンジ） | 太さ 3 px |
| スポット | `#1E90FF`（青） | 一辺 5 px |
| トラック | `#000080`（ネイビー） | 太さ 4 px |
| 写真撮影場所 | `#000080`（ネイビー） | 半径 6 px |

カラーピッカーまたは `#RRGGBB` 形式の文字列で色を、数値入力でサイズを変更できます。**[設定]** をタップすると、表示中のレイヤーが即座に再描画されます。

### 15.4 過去データを Drive に移行（管理者向け）

「▶」ボタンで展開し、Firestore `tracks` コレクションのドキュメント一覧を取得します。前方一致テキストで絞り込み、複数選択して **[実行]** をタップすると、選択したルートを Google Drive へ移行します（写真・サムネール・KMZ がアップロードされます）。

## 17. よくある質問

### Q1. GPS精度が悪いのですが？

**A:** 以下をお試しください。
- 建物の外、空が開けた場所に移動する
- 端末の位置情報設定を「高精度」にする
- しばらく待つ（GPS 補足に時間がかかる場合があります）

### Q2. 写真が保存されません

**A:** GPS 記録中（Start 後）でないと写真は撮影できません。先に **Start** を押してから撮影してください。

### Q3. Save ボタンで KMZ が保存されず、Firebase に繋ごうとします

**A:** 設定（⚙）の「保存時にクラウドのデータベースを使用」が ON になっています。OFF にすると KMZ ファイルとして保存されます。ダイアログタイトルが「Save to cloud as...」ならクラウド保存、「Save to file as...」ならファイル保存です。

### Q4. クラウド保存ができません

**A:**
- インターネット接続を確認
- 設定の「Use Firebase」が ON になっているか確認
- 設定の「ユーザー接続」エリアでユーザー名・メールを登録しているか確認
- 写真が多い場合はアップロードに時間がかかります

### Q5. KMZ メールが届きません

**A:** クラウド保存に成功すると、サーバー側で KMZ を生成して登録メールアドレスへ送信します。次の点をご確認ください。
- 設定の「ユーザー接続」エリアでメールアドレスを登録しているか
- 迷惑メールフォルダに入っていないか
- ステータスに `KMZ送信失敗: ...` と表示されていないか

### Q6. 過去のデータが消えました

**A:** ブラウザのデータを削除すると、ローカルデータ（IndexedDB）は消去されます。重要なデータは必ず Save（KMZ またはクラウド）してください。クラウド保存していれば、Cloud Function が KMZ をメールで送信しているのでメールから復旧できます。

### Q7. バッテリーの消費が激しいです

**A:** GPS 追跡中は位置情報を常時取得するため、バッテリー消費が増加します。長時間記録時はモバイルバッテリーを携帯することを推奨します。

### Q8. 方向ボタン（Forward/Backward）が表示されません

**A:** 設定の「進行方向ボタンを表示」が OFF になっています。ON にしてください。

### Q9. メモを編集中に別の写真へ切り替えたら確認ダイアログが出ました

**A:** 編集途中の変更を「保存して移動」するか「変更を破棄して移動」するかを選んでください。

### Q10. 写真ビューアから写真を削除できますか

**A:** はい。写真ビューアの **Delete** ボタンで削除できます。確認ダイアログのあと、写真と地図上のマーカーが削除されます。最後の1枚を削除するとビューアが閉じます。

### Q11. ポケットの中で iPhone を振ったら「元に戻す」ダイアログが表示された

**A:** iOS の「Shake to Undo」機能によるものです。RouteLogger は記録開始時・ダイアログ閉鎖時・画面非表示時にテキスト入力のフォーカスと履歴を解除して、このダイアログが表示されにくくなるよう対策しています。完全に防止したい場合は、iOS の **設定** → **アクセシビリティ** → **タッチ** → **「振って元に戻す」** を OFF にしてください。

### Q12. 隠し設定の「画面設定等」はどうやって開きますか

**A:** Settings ダイアログの右下にある **アプリバージョン** ラベルを 3秒以内に 5回連続タップしてください。

### Q13. 外部写真の「元の写真を表示」をタップしても画像が出ません

**A:** Google Drive 側のアクセス権限（共有設定）が「リンクを知っている全員」になっているか確認してください。直接画像が読めない場合は、自動で埋め込みプレビュー（iframe）に切り替わります。

## 18. トラブルシューティング

### 18.1 位置情報が取得できない

1. ブラウザの位置情報許可を確認
2. 端末の位置情報サービスが ON か確認
3. 屋外の開けた場所で再試行

### 18.2 カメラが起動しない

1. ブラウザのカメラ許可を確認
2. 他のアプリがカメラを使用していないか確認
3. ページを再読み込み

### 18.3 画面が固まった

1. ページを再読み込み（更新）
2. ブラウザを一度閉じて再起動
3. 端末を再起動

### 18.4 KMZ/GeoJSONファイルが読み込めない

1. ファイルの拡張子が `.kmz`, `.kml`, `.geojson`, `.json`, `.zip` のいずれかか確認
2. ファイルが破損していないか確認
3. ブラウザを再読み込みして再試行

### 18.5 アプリの新バージョンが反映されない

サーバーから新バージョンが配信されると、Service Worker がページを自動的にリロードします。アプリバージョン（Settings の右下）が変わらない場合は、一度ブラウザを完全に閉じて再起動してください。
